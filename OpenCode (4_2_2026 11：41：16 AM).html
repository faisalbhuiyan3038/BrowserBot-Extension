2. ASK PAGE IMPLEMENTATION

The Ask Page is a chat overlay that appears on any web page, allowing users to ask AI questions about the current page content.

Entry Point: M:\.temp\BrowserBot-Extension\entrypoints\ask-page.content\index.tsx

This is a content script defined via WXT's defineContentScript:

  • matches: ['<all_urls>'] -- injected on every page
  • cssInjectionMode: 'ui' -- uses WXT's createShadowRootUi for style isolation
  • Mounting flow:
    1. Listens for TOGGLE_ASK_PAGE messages from the background script
    2. On first toggle, calls mountUI() which creates a Shadow DOM overlay via createShadowRootUi()
    3. The Shadow DOM UI is named browserbot-ask-page, positioned as overlay, with zIndex: 2147483646 (max)
    4. React renders AskPagePanel into #browserbot-ask-page-root inside the shadow root
    5. Styles are injected from getStyles() (from chatStyles.ts) into the shadow root
    6. On subsequent toggles, it calls the registered showCallback to un-minimize the panel
    7. Also listens for EXTRACT_PAGE_CONTENT messages and responds with extracted content

Main Component: M:\.temp\BrowserBot-Extension\entrypoints\ask-page.content\AskPagePanel.tsx (960 lines)

This is the core React component. Key features:

State:

  • minimized -- collapsed to a floating tab button
  • messages -- chat message array (ChatMsg[])
  • input -- textarea value
  • isStreaming -- whether AI is currently responding
  • panelWidth -- resizable width (320-800px, default 420)
  • persistChat -- whether to persist chat across page navigations
  • currentTabAttached -- whether current page content is attached as context
  • attachedTabs -- array of other tabs attached as context (AttachedTab[])
  • providerType -- 'openai' | 'ollama' | 'chrome_ai'
  • openaiProviders -- list of configured OpenAI-compatible providers
  • selectedOpenAIId -- active OpenAI provider ID
  • ollamaModel -- current Ollama model
  • systemPrompt -- always-active system prompt with variable placeholders
  • quickPrompts -- reusable prompt templates
  • thinkingContent / thinkingExpanded -- reasoning/thinking display
  • showHistory / conversations / activeConversationId -- chat history
  • extractionAlgorithm -- 1=Text, 2=Optimized, 3=Full

Key Functions:

  • sendMessage() -- Builds the full prompt (system prompt + variable interpolation + tab context + conversation history), sends ASK_PAGE_CHAT message to background
  • openTabPicker() / confirmTabSelection() -- Fetches tab list via GET_TAB_LIST, then extracts content via GET_TAB_CONTENT
  • saveCurrentConversation() -- Debounced auto-save of conversation to ConversationStorage
  • renderMarkdown() -- Uses marked library, injects "Copy" buttons into code blocks
  • abortStream() -- Sends ASK_PAGE_CHAT_ABORT to stop streaming

Variable Interpolation in System Prompt: The system prompt supports these placeholders (replaced at send time):

  • {pageTitle} -- current page title
  • {pageUrl} -- current page URL
  • {pageContent} -- extracted page content (only extracted if placeholder is present)
  • {selectedText} -- user's current text selection
  • {tabContext} -- content from attached tabs

Streaming Chunk Listener: Listens for these message types from background:

  • ASK_PAGE_CHAT_CHUNK -- appends chunk to last assistant message
  • ASK_PAGE_CHAT_THINKING -- accumulates thinking/reasoning content
  • ASK_PAGE_CHAT_DONE -- finalizes thinking, sets isStreaming = false
  • ASK_PAGE_CHAT_ERROR -- adds error message
  • CHAT_UPDATED -- real-time sync from other tabs (when persistChat is on)

  truncatedLength: number,   // length after truncation
  algorithm: number          // which algorithm was used

Page Content Extraction: M:\.temp\BrowserBot-Extension\utils\extractor.ts (684 lines)

Three algorithms, selected via pageExtractionAlgorithm storage setting:

Algorithm 1 -- Text Extraction (default):

  • Lightweight DOM traversal: extracts headings (h1-h6 as markdown), paragraphs, list items, table cells
  • Ignores nav, aside, header, footer, buttons, scripts, styles, forms
  • Smart truncation: takes first 40% of content, then samples proportionally from the rest
  • Character limit: 20,000 (configurable)
  • No external dependencies

Algorithm 2 -- Optimized Content Extraction:

  • Full HTML cleaning via harpaExtractPageText(): strips scripts/styles, removes empty elements, deduplicates lines
  • YouTube transcript extraction when on youtube.com/watch or youtube.com/shorts/
  • Multiple transcript extraction strategies: timedtext API, transcript panel scraping, YouTube Data API fallback
  • Same smart truncation as Algorithm 1

Algorithm 3 -- Full Content Extraction:

  • Uses @mozilla/readability (Readability API) for article extraction
  • Uses html-to-text for HTML-to-text conversion
  • Clones DOM, strips invisible elements, scripts, styles, SVGs, iframes, forms
  • Returns both article content and full page text
  • Dynamic imports to avoid bundling these heavy deps for algorithms 1/2

4. MESSAGING / ROUTING

Background Script Message Router: M:\.temp\BrowserBot-Extension\entrypoints\background.ts

The background script is the central message hub. All message types and their handlers:

Message Type Direction Handler Description
OPEN_CHAT_TAB Any -> Background Creates new tab at /chat.html Opens fullscreen chat
TOGGLE_ASK_PAGE Popup/Any -> Background -> Content Script Queries active tab, forwards TOGGLE_ASK_PAGE with pageTitle/pageUrl Toggles overlay
ASK_PAGE_CHAT Content Script/Chat Page -> Background handleAskPageChat() -- streams AI response with chunk dispatch Main chat request
ASK_PAGE_CHAT_ABORT Content Script/Chat Page -> Background Removes abort listener, signals abort controller Stop streaming
GET_TAB_LIST Content Script -> Background handleGetTabList() -- returns filtered tab array Tab picker data
GET_TAB_CONTENT Content Script -> Background handleGetTabContent() -- messages content script or injects script fallback Extract tab content
SAVE_PANEL_WIDTH Content Script -> Background Saves askPagePanelWidth to AppStorage Persist width
SAVE_CHAT Content Script -> Background Saves to browser.storage.session key askPageChat, broadcasts CHAT_UPDATED Persist chat
LOAD_CHAT Content Script -> Background Reads askPageChat from session storage Load persisted chat
CLEAR_CHAT Content Script -> Background Removes askPageChat from session storage, broadcasts update Clear chat
SAVE_CONVERSATION Content Script -> Background Saves via ConversationStorage.save() Save conversation
LOAD_CONVERSATIONS Content Script -> Background Loads all via ConversationStorage.loadAll() Load history
DELETE_CONVERSATION Content Script -> Background Deletes via ConversationStorage.delete() Delete conversation
CHECK_CHROME_AI Options Page -> Background Calls checkChromeAIStatus() Check Chrome AI
DOWNLOAD_CHROME_AI Options Page -> Background Calls handleDownloadChromeAI() with progress updates Download Chrome AI model

Dispatch Patterns:

Streaming chunks are dispatched via dispatchChunk() which routes to either:

  • browser.runtime.sendMessage() -- for extension pages (chat tab)
  • browser.tabs.sendMessage(tabId) -- for content scripts

Broadcast to all tabs via broadcastToTabs():

  • Used for CHAT_UPDATED to sync persisted chat across tabs
  • Excludes the sender's tab ID

Abort mechanism:

  • A temporary abortListener is registered on browser.runtime.onMessage during streaming
  • Matches by sessionId for extension pages, or by tabId for content scripts
  • Cleaned up in finally block

Detailed Communication Flows:

Flow 1: Toggle Ask Page (Popup -> Background -> Content Script)

  1. User clicks "Ask Page" in popup
  2. Popup sends browser.runtime.sendMessage({ type: 'TOGGLE_ASK_PAGE' })
  3. Background receives it, queries active tab, sends browser.tabs.sendMessage(tabId, { type: 'TOGGLE_ASK_PAGE', pageTitle, pageUrl })
  4. Content script receives it, mounts the Shadow DOM UI via createShadowRootUi

Flow 2: Chat Request (Content Script -> Background -> AI Provider -> Content Script)

  1. User types message and sends in AskPagePanel
  2. Panel sends browser.runtime.sendMessage({ type: 'ASK_PAGE_CHAT', messages, providerType, openaiProviderId, sessionId })
  3. Background's handleAskPageChat() calls streamChatWithAI() from askPageAI.ts
  4. AI provider streams response; each chunk is dispatched back:
    • For content scripts: browser.tabs.sendMessage(tabId, { type: 'ASK_PAGE_CHAT_CHUNK', chunk, sessionId })
    • For extension pages: browser.runtime.sendMessage({ type: 'ASK_PAGE_CHAT_CHUNK', chunk, sessionId })
  5. Content script's chunk listener appends to the UI in real-time

Flow 3: Tab Content Extraction (Content Script -> Background -> Content Script)

  1. User clicks "Add Tab" in AskPagePanel, selects a tab
  2. Panel sends browser.runtime.sendMessage({ type: 'GET_TAB_CONTENT', tabId })
  3. Background sends browser.tabs.sendMessage(tabId, { type: 'EXTRACT_PAGE_CONTENT', algorithm }) to the target tab's content script
  4. Target content script runs extractPageContent(algorithm) and returns result
  5. If content script not present, background falls back to browser.scripting.executeScript() injecting document.body.innerText.substring(0, 20000)

Flow 4: Tab List (Content Script -> Background)

  1. Panel sends browser.runtime.sendMessage({ type: 'GET_TAB_LIST' })
  2. Background queries all tabs, filters out chrome://, about:, chrome-extension://, moz-extension:// URLs
  3. Returns array of {id, title, url, favIconUrl}

Flow 5: Chat Persistence (Content Script <-> Background <-> Other Tabs)

  1. When persistChat is true, every message change triggers browser.runtime.sendMessage({ type: 'SAVE_CHAT', messages })
  2. Background saves to browser.storage.session.set({ askPageChat: messages })
  3. Background broadcasts CHAT_UPDATED to all tabs except sender via broadcastToTabs()
  4. Other AskPagePanel instances receive CHAT_UPDATED and sync their message state

Flow 6: Settings Sync (Storage onChanged)

  1. Options page calls AppStorage.set(patch) which writes to browser.storage.local
  2. AskPagePanel listens to browser.storage.onChanged and updates its local state when appState changes
  3. This ensures settings changes in the options page are reflected instantly in the Ask Page panel

Flow 7: Chrome AI Download (Options Page -> Background -> Options Page)

  1. Options page sends browser.runtime.sendMessage({ type: 'DOWNLOAD_CHROME_AI' })
  2. Background calls downloadChromeAIModel() which registers a progress monitor
  3. Progress updates sent via browser.tabs.sendMessage(tabId, { type: 'CHROME_AI_DOWNLOAD_PROGRESS', progress })
  4. Options page listens for progress and updates its progress bar
